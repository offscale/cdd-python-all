# Publishing the Generated Client Output

This document details how to automate the publication of the Python Client Library generated by `cdd-python`, ensuring it stays in sync with its corresponding OpenAPI server specifications.

## The Goal

When the upstream server (OpenAPI `openapi.json`) changes, the Python client repository should automatically detect the change, regenerate the client code, update the library version, and publish the new client back to PyPI.

## Automated Updating (Cronjob)

You can set up a GitHub Action to fetch the latest OpenAPI spec on a schedule or via a webhook triggered by the server repository.

### Example GitHub Action (`.github/workflows/update_client.yml`)

```yaml
name: Update Client Library

on:
  schedule:
    - cron: '0 0 * * *' # Run daily at midnight
  workflow_dispatch: # Allow manual triggers
  repository_dispatch:
    types: [openapi-updated] # Allow trigger from server repository

jobs:
  update-client:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'

      - name: Install dependencies
        run: pip install build twine cdd-python

      - name: Fetch Latest OpenAPI Spec
        run: curl -sL https://api.yoursite.com/openapi.json -o openapi.json

      - name: Generate Client
        run: cdd-python from_openapi -i openapi.json -o src/

      - name: Check for Changes
        id: git-check
        run: |
          git diff --exit-code || echo "changed=true" >> $GITHUB_OUTPUT

      - name: Commit and Push Update
        if: steps.git-check.outputs.changed == 'true'
        run: |
          git config --global user.name "github-actions[bot]"
          git config --global user.email "github-actions[bot]@users.noreply.github.com"
          git add src/ openapi.json
          git commit -m "Auto-update client from OpenAPI spec"
          git push

      - name: Bump Version and Build
        if: steps.git-check.outputs.changed == 'true'
        run: |
          # Use bump2version or a custom script to bump the version in pyproject.toml
          python -m build

      - name: Publish to PyPI
        if: steps.git-check.outputs.changed == 'true'
        env:
          TWINE_USERNAME: __token__
          TWINE_PASSWORD: ${{ secrets.PYPI_API_TOKEN }}
        run: twine upload dist/*
```

## Manual Updating

To manually sync the client library and publish:

1. Fetch the latest `openapi.json` from the server.
2. Run `cdd-python from_openapi -i openapi.json -o src/`
3. Review changes: `git diff`
4. Commit: `git commit -m "chore: sync with latest OpenAPI spec"`
5. Bump version in `pyproject.toml`
6. Build: `make build`
7. Publish: `twine upload dist/*`
